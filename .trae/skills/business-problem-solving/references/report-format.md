# 报告输出格式

完成四步分析后，按以下结构输出报告。

```
# 问题分析报告

## 1. 差距分析

### 主题
[问题描述]

### As is (现状)
[现状描述]

### To be (理想)
[理想状态描述]

### Gap (落差)
[差距描述]

---

## 2. 6W2H检视

| 维度 | 回答 |
|---|---|
| Who | [回答] |
| What | [回答] |
| When | [回答] |
| Where | [回答] |
| Why | [回答] |
| Which | [回答] |
| How | [回答] |
| How Much | [回答] |

---

## 3. 5Why分析

```
问题: [问题描述]
  ↓ Why?
原因1: [原因描述]
  ↓ Why?
原因2: [原因描述]
  ↓ Why?
原因3: [原因描述]
  ↓ Why?
原因4: [原因描述]
  ↓ Why?
原因5: [原因描述] (根本原因)
```

---

## 4. 可控性分析

| 原因 | 可控性 | 对策/行动 |
|---|---|---|
| 原因1 | [可控制/不可控制] | [对策内容] |
| 原因2 | [可控制/不可控制] | [对策内容] |
| 原因3 | [可控制/不可控制] | [对策内容] |
| 原因4 | [可控制/不可控制] | [对策内容] |
| 原因5 | [可控制/不可控制] | [对策内容] |
```

---

## 报告命名格式

```
problem_solving_report_YYYYMMDD_HHMMSS.md
```

示例：`problem_solving_report_20260126_111252.md`
