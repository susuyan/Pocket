# 管理組織

## 任务目标
運用三步驟方法论幫助管理者系統化建立高效團隊，從目標共享到關係建立再到激勵機制，形成可持續發展的組織文化。

## 触发方式

### 主动触发
当用户提出以下类型的需求时触发：
- "管理团队"
- "建立团队文化"
- "提升团队效率"
- "激励员工"
- 系統化组织管理

### 命令触发
```bash
/manage-team [團隊主題]
```

## 核心原則：開會前先制定基本規範

### 团队基本规范

在開始任何組織管理活動前，必須先確立團隊基本規範：

| 規範類型 | 內容 | 範例 |
|---------|------|------|
| 溝通規範 | 如何溝通 | 開誠布公、積極傾聽 |
| 決策規範 | 如何做決定 | 共識決策、投票投票 |
| 會議規範 | 如何開會 | 準時出席、提前準備 |
| 反饋規範 | 如何給反饋 | 三明治法、SBI feedback |
| 衝突規範 | 如何處理衝突 | 聚焦議題、對事不對人 |

**建立方式**：
1. 召集團隊成員
2. 開放討論規範內容
3. 形成共識後記錄
4. 全體確認遵循

## STEP 1: 全體清楚掌握目標

目标：確保所有成員對組織目標有共同理解。

### 1.1 任務．願景．價值

**任務（Mission）**
- 定義：我們是誰？我們做什麼？
- 問題：
  - 我們的業務是什麼？
  - 我們的客戶是誰？
  - 我們如何創造價值？

**願景（Vision）**
- 定義：我們想要到達哪里？未來樣貌？
- 問題：
  - 3-5年後我們想成為什麼樣的團隊？
  - 我們想達成什麼成就？

**價值（Values）**
- 定義：我們相信什麼？如何做決策？
- 問題：
  - 什麼對我們最重要？
  - 我們的行為準則是什麼？

**範例**：
```
任務：打造最優質的產品體驗
願景：成為行業領導者，改變用戶的生活方式
價值：誠信、創新、用戶導向、協作
```

### 1.2 Will / Can / Must

核心：以三個維度評估任務或目標的適配性。

| 維度 | 含義 | 問問自己 |
|------|------|---------|
| Will | 想不想 | 這是我想要做的嗎？ |
| Can | 能不能 | 我有能力做到嗎？ |
| Must | 需不需要 | 這是需要做的事嗎？ |

**矩陣分析**：

| | 不需要 (Must: No) | 需要 (Must: Yes) |
|----|------------------|-----------------|
| 不想 (Will: No) | 無意義的任務 | 應該拒絕 |
| 想 (Will: Yes) | 有趣但優先不高 | 真正的核心任務 |

**應用**：
- Will √ Can √ Must √ = 完美任務，全力以赴
- Will ✗ 就算能做也需要重新評估
- Must ✗ 即使想做也需要重新排列

### 1.3 Need / Want 矩陣

核心：區分需求與欲望，優化資源分配。

| | Need (需要) | Want (想要) |
|----|-------------|-------------|
| 高重要 | A. 優先處理 | B. 平衡處理 |
| 低重要 | C. 擇機處理 | D. 捨棄 |

**分類說明**：

**A. 優先處理（需要 + 高重要）**
- 必不可少的業務需求
- 影響核心目標
- 資源集中投入

**B. 平衡處理（想要 + 高重要）**
- 高價值的增強功能
- 競爭优势來源
- 適當資源分配

**C. 擇機處理（需要 + 低重要）**
- 基礎維護工作
- 低成本優化
- 利用閒置資源

**D. 捨棄（想要 + 低重要）**
- 非核心、非必要
- 浪費資源
- 直接剔除

## STEP 2: 提昇成員關係的品質

目标：建立信任、暢通溝通的團隊關係。

### 2.1 周哈里窗（Johari Window）

核心：認識自我與他人的四個象限。

```
            自己知道           自己不知道
        ┌───────────┐        ┌───────────┐
他人知道 │  開放區   │  盲區  │
 (Arena)  │  (Open)   │ (Blind)│
        └───────────┘        └───────────┘
        ┌───────────┐        ┌───────────┐
他人不知道│  隱藏區   │  未知區  │
  (Hidden)│  (Hidden) │(Unknown)│
        └───────────┘        └───────────┘
```

**象限說明**：

| 象限 | 說明 | 擴大方式 |
|------|------|---------|
| 開放區 | 自己和他人都知道 | 誠實溝通、主動分享 |
| 盲區 | 他人知道，自己不知道 | 接受反饋、自我反思 |
| 隱藏區 | 自己知道，他人不知道 | 開放分享、降低防禦 |
| 未知區 | 自己和他人都不知道 | 共同探索、嘗試 |

**應用原則**：
- 擴大開放區：增加了解與信任
- 縮小盲區：接受他人反饋
- 縮小隱藏區：主動分享信息
- 探索未知區：共同學習成長

### 2.2 認知／行動循環

核心：理解改進的循環過程。

```
┌─────────┐
│  現況   │
└────┬────┘
     │ 問：「為什麼這樣？」
     ▼
┌─────────┐
│  認知   │─→ 意識到問題
└────┬────┘
     │ 問：「實際狀況如何？」
     ▼
┌─────────┐
│  事實   │
└────┬────┘
     │ 問：「想怎樣？」
     ▼
┌─────────┐
│  目標   │
└────┬────┘
     │ 問：「做什麼？」
     ▼
┌─────────┐
│  行動   │
└────┬────┘
     │ 問：「結果如何？」
     ▼
   現況 (回到起始點)
```

**應用要點**：
- 每個階段問對問題
- 循環持續進行
- 事實與目標對照

### 2.3 Want / Commitment

核心：區分「想要」與「承諾」。

| 狀態 | 含義 | 行為特徵 |
|------|------|---------|
| Want | 想要、期望 | 口頭支持，行動不穩 |
| Commitment | 承諾 | 明確約定，持續行動 |

**W/C 矩陣**：

| | 無 Want | 有 Want |
|----|---------|---------|
| 無 Commitment | 無意也無承諾 | 想但不承諾 |
| 有 Commitment | 責任驅動 | 完美狀態 |

**轉化策略**：
- Want √, Commitment ✗：建立約定機制
- Want ✗, Commitment √：溝通價值、激發動機

### 2.4 PM理論（Positive Mentoring）

核心：正向導師關係管理。

**三個要素**：

| 要素 | 說明 | 行動 |
|------|------|------|
| P (Positive) | 正面態度 | 專注優點、給予鼓勵 |
| M (Model) | 模範行為 | 以身作則、展示期待 |
| (Mentoring) | 導師關係 | 指導成長、建立信任 |

**應用方法**：
- 發現成員優點並強化
- 以自己的行為作為示範
- 建立一對一的對話機制
- 提供具體的反饋與指導

### 2.5 利害關係人分析

核心：識別並管理所有利害關係人。

**分析框架**：

| 利害關係人 | 影響程度 | 利益相關性 | 關注重點 | 應對策略 |
|-----------|---------|-----------|---------|---------|
| 客戶 | 高 | 高 | 產品品質、服務 | 溝通滿意度 |
| 員工 | 高 | 高 | 待遇、發展 | 透明溝通 |
| 投資人 | 高 | 高 | ROI、成長 | 定期報告 |
| 供應商 | 中 | 中 | 合作穩定 | 建立夥伴 |
| 政府 | 中 | 中 | 法規遵循 | 主動配合 |

**應用步驟**：
1. 列出所有利害關係人
2. 評估影響程度與利益相關性
3. 識別各自的關注重點
4. 制定應對策略

## STEP 3: 提昇成員的動機

目标：激發成員主動性與持續動力。

### 3.1 雙因素理論

由来：赫茲伯格提出激勵與保健兩類因素。

**兩類因素**：

| 類型 | 說明 | 範例 | 作用 |
|------|------|------|------|
| 保健因素 | 避免不滿意 | 工作環境、薪酬、安全 | 沒有會不滿，有不一定滿意 |
| 激勵因素 | 增加滿意意 | 成就、認可、成長 | 有會滿意，加強動機 |

**應用原則**：
- 先确保保健因素：避免不滿意
- 再加入激勵因素：提升滿意度
- 兩者並重才能建立持續動機

### 3.2 Will / Skill 矩陣

核心：依據意願與能力進行管理。

```
                高意願
            ┌─────────┐
            │ 明星型  │
            │ 放手做  │
    ┌───────┼─────────┤
 低能力│  跟進型│ 明星型  │
            │ 培訓關注│ 放手做  │
    └───────┼─────────┤
            │ 責任型  │
            │ 放手做  │
            └─────────┘
            低意願
```

| 類型 | 能力 | 意願 | 管理方式 |
|------|------|------|---------|
| 明星型 | 高 | 高 | 放手、授權、挑戰 |
| 跟進型 | 低 | 高 | 培訓、輔導、支持 |
| 責任型 | 高 | 低 | 驗證、確認、激勵 |
| 問題型 | 低 | 低 | 評估、決策、處理 |

### 3.3 GROW 模型

核心：一對一導師輔導的結構化方法。

| 字母 | 英文 | 中文 | 問題 |
|------|------|------|------|
| G | Goal | 目標 | 「你想要達成什麼？」 |
| R | Reality | 現實 | 「目前的情況如何？」 |
| O | Options | 方案 | 「有哪些可行的方案？」 |
| W | Will | 行動 | 「你打算怎麼做？」 |

**應用範例**：

**Goal（目標）**
- 「下一個月你想要達成什麼目標？」

**Reality（現實）**
- 「目前的數據如何？」
- 「什麼在阻礙你？」

**Options（方案）**
- 「你能做什麼？」
- 「誰能幫助你？」

**Will（行動）**
- 「你打算做哪個方案？」
- 「什麼時候開始？」

## 完整工作流程

```
開始
 ↓
制定團隊基本規範
 ├─ 溝通規範
 ├─ 決策規範
 ├─ 會議規範
 ├─ 反饋規範
 └─ 衝突規範
 ↓
STEP 1: 全體清楚掌握目標
 ├─ 任務．願景．價值
 ├─ Will / Can / Must
 └─ Need / Want 矩陣
 ↓
STEP 2: 提昇成員關係品質
 ├─ 周哈里窗
 ├─ 認知／行動循環
 ├─ Want / Commitment
 ├─ PM理論
 └─ 利害關係人分析
 ↓
STEP 3: 提昇成員動機
 ├─ 雙因素理論
 ├─ Will / Skill 矩陣
 └─ GROW模型
 ↓
生成組織管理報告
 ↓
執行與持續改進
 ↓
結束
```

## 关键原则

- **共識優先**：目標和規範需要全體共識
- **溝通開放**：建立安全的溝通環境
- **信任為本**：團隊關係基於信任
- **激勵持續**：動機需要持續維護
- **循環改進**：定期檢討與調整
- **榜樣帶動**：管理者以身作則
