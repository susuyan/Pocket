# 方法论详解

## STEP 1: 掌握目标

### 1.1 任務．願景．價值

#### 起源
源自彼得·德魯克（Peter Drucker）的組織管理理論。

#### 任務（Mission）

**定義**：
組織存在的基本目的，回答「我們是誰」和「我們做什麼」。

**制定原則**：
- 清晰簡潔：能被所有人理解
- 行動導向：描述具體行動
- 差異化：與競爭者區分

**範例對比**：

| 差 | 良 | 優 |
|----|----|-----
| 我們做軟體 | 我們開發企業軟體解決方案 | 我們通過創新技術幫助企業提升效率 |
| 我們賣咖啡 | 我們提供高品質咖啡產品 | 我們連接人們，創造美好咖啡體驗 |

**制定問句**：
- 我們的業務是什麼？
- 我們的客戶是誰？
- 我們如何創造價值？
- 我們的產品/服務解決什麼問題？

---

#### 願景（Vision）

**定義**：
組織期望達到未來狀態的畫像，回答「我們想去哪裡」。

**時間範圍**：
- 短期願景：1-2年
- 中期願景：3-5年
- 長期願景：5-10年

**制定原則**：
- 有啟發性：激勵人心
- 可視化：能形成畫像
- 挑戰性：需要努力才能達成

**範例對比**：

| 差 | 良 | 優 |
|----|----|----|
| 我們想更強大 | 成為行業領導者 | 十年後，成為全球最受信賴的技術創新公司 |
| 我們想賺更多錢 | 持續成長 | 改變人們的工作方式，讓每個人都能發揮潛力 |

**制定問句**：
- 3-5年後我們希望成為什麼樣的組織？
- 我們想要達成什麼重大成就？
- 如果我們成功了，世界會有什麼不同？

---

#### 價值（Values）

**定義**：
組織相信的原則和行為準則，回答「我們相信什麼」和「我們如何做決策」。

**制定原則**：
- 清晰明確：每個人都懂
- 行為導向：可實踐
- 一致性：所有決策都符合

**常見價值範例**：

| 類別 | 範例 |
|------|------|
| 誠信 | 誠實正直、透明開放 |
| 客戶導向 | 客戶第一、用心服務 |
| 創新 | 勇於創新、持續改進 |
| 協作 | 團隊合作、互相支持 |
| 品質 | 追求卓越、精益求精 |
| 責任 | 承擔責任、結果導向 |

**制定問句**：
- 什麼對我們最重要？
- 我們如何做決策？
- 我們期望的行為準則是什麼？
- 什麼是我們絕對不會做的？

---

#### MVP 一體化範例

**科技公司範例**：

```
任務：打造革命性的產品體驗，讓每個人的工作更簡單

願景：成為全球最受信賴的工作效率平台，
      改變十億人的工作方式

價值：
  • 用戶導向 - 始終從用戶需求出發
  • 創新精神 - 敢於挑戰現状
  • 協作共贏 - 內外團隊合作
  • 持續成長 - 永不放棄學習
```

### 1.2 Will / Can / Must

#### 核心概念

以三個維度評估任務或目標的適配性，幫助決策優先順序。

#### 三個維度

| 維度 | 中文 | 含義 | 詢問問題 |
|------|------|------|---------|
| Will | 想不想 | 內在動機、激勵程度 | 這是我想要做的嗎？ |
| Can | 能不能 | 能力、資源、可行性 | 我有能力做到嗎？ |
| Must | 需不需要 | 緊急性、重要性 | 這是現階段需要做的事嗎？ |

#### 矩陣分析

```
                  Must (需要)
            ┌─────────────────┐
            │  優先核心任務   │
  Will (想) │                 │
            │  全力以赴      │
    ┌───────┼─────────────────┤
    │       │  需要拒絕的     │
    │       │      任務       │
    └───────┼─────────────────┘
            └─────────────────┘
             不需要 (Must: 否)
```

#### 八種組合與應對策略

| Will | Can | Must | 說明 | 策略 |
|------|-----|------|------|------|
| √ | √ | √ | 想做、能做、需要做 | **最優先**：全力以赴 |
| √ | √ | ✗ | 想做、能做、不需要 | **延後**：儲備未來 |
| √ | ✗ | √ | 想做、不能做、需要做 | **培訓**：提升能力或尋求支援 |
| √ | ✗ | ✗ | 想做、不能做、不需要 | **擇機**：有機會再嘗試 |
| ✗ | √ | √ | 不想做、能做、需要做 | **溝通**：解釋價值，建立承諾 |
| ✗ | √ | ✗ | 不想做、能做、不需要 | **拒絕**：拒絕或委託 |
| ✗ | ✗ | √ | 不想做、不能做、需要做 | **緊急**：立刻處理或尋求外部幫助 |
| ✗ | ✗ | ✗ | 不想做、不能做、不需要 | **無視**：不投入資源 |

#### 應用場景

**個人任務優先**：
- 每天開始前列出任務清單
- 為每個任務評估 W/C/M
- 優先處理 √/√/√ 任務

**團隊目標設定**：
- 團隊成員共同評估每個目標
- 確保每個人對 W/C/M 有共識
- 根據評估結果分配資源

**專案決策**：
- 新專案提案時評估
- 比較不同專案的 W/C/M
- 選擇最優專案執行

### 1.3 Need / Want 矩陣

#### 核心概念

區分需求（Need）與欲望（Want），透過重要性質化進行優化。

#### 矩陣結構

```
            高重要性
        ┌─────────────┐
        │   A. 優先   │
  Want  │     處理    │
(欲望)  │             │
    ┌───┼─────────────┤
    │   │   B. 平衡   │
    │   │     處理    │
    └───┼─────────────┘
        └─────────────┘
  低重要           高重要
        Need (需要)
```

#### 四象限說明

**A. 優先處理想（Want + 高重要）**
- 定義：想要且高重要
- 特點：競爭優勢、高價值
- 策略：重點投資、快速反應

**B. 平衡處理想（Need + 高重要）**
- 定義：需要且高重要
- 特點：核心功能、業務需求
- 策略：平衡品質與效率

**C. 擇機處理想（Need + 低重要）**
- 定義：需要但低重要
- 特點：基礎維護、優化需求
- 策略：機會允許時處理

**D. 捨棄想（Want + 低重要）**
- 定義：想要但低重要
- 特點：非核心、低價值
- 策略：直接捨棄

#### 應用示例

**產品功能優先**

| 功能 | Need | Want | 重要性 | 分類 |
|------|------|------|--------|------|
| 用戶登入 | √ | - | 高 | B |
| 暗黑模式 | √ | √ | 中 | B |
| 主題遊戲 | - | √ | 低 | D |
| 效能優化 | √ | √ | 高 | A |
| 社群分享 | √ | - | 中 | C |

**個人時間管理**

| 活動 | Need | Want | 重要性 | 分類 |
|------|------|------|--------|------|
| 重要客戶會議 | √ | √ | 高 | A |
| 行程整理 | √ | - | 低 | C |
| 社交媒體瀏覽 | - | √ | 低 | D |
| 團隊培訓 | √ | √ | 高 | A |
| 午休散步 | - | √ | 中 | B |

## STEP 2: 成員關係品質

### 2.1 周哈里窗（Johari Window）

#### 起源
由 Joseph Luft 和 Harry Ingham 於 1955 年提出。

#### 四個象限

```
            自己知道           自己不知道
        ┌───────────┐        ┌───────────┐
他人知道 │  開放區   │  盲 區  │
 (Arena)  │  (Open)   │ (Blind)│
        └───────────┘        └───────────┘
        ┌───────────┐        ┌───────────┐
他人不知道│  隱藏區   │  未知區  │
  (Hidden)│  (Hidden) │(Unknown)│
        └───────────┘        └───────────┘
```

#### 象限詳細說明

**開放區（Arena）**
- 定義：自己和他人都了解的區域
- 例子：大家都知道你喜歡咖啡、擅長Python、專業領域是後端開發
- 擴大方式：主動分享、誠實溝通
- 價值：建立信任、高效協作

**盲區（Blind）**
- 定義：別人知道、自己不知道的區域
- 例子：你有講話太快的習慣、你的提案格式可以改進
- 縮小方式：接受反饋、觀察他人反應、積極請教
- 價值：改善自我、了解影響

**隱藏區（Hidden）**
- 定義：自己知道、別人不知道的區域
- 例子：你不喜歡公開演講、你正在學習新技術、你對某項決策有保留
- 縮小方式：適當分享、降低防禦、建立信任
- 價值：增加理解、協同解決問題

**未知區（Unknown）**
- 定義：自己和他都不知道的區域
- 例子：你還未發現的核心能力、潛在的創新機會
- 探索方式：共同嘗試、實驗學習、開放心態
- 價值：發現新可能、共同成長

#### 團隊應用

**新團隊建立期**：
```
             ◇大◇
             ◇盲◇      ◇隱◇
             ◇區◇      ◇藏◇
        │←──┼──────┼──→│
         ◇開◇大      ◇區◇
         ◇放◇      │
         ◇區◇      ◇未◇
                   ◇知◇
```

特點：开放区小，隐藏区和盲区大

**行動**：
- 破冰活動
- 自我介紹
- 建立反饋文化

**成熟團隊穩定期**：
```
             ◇小◇
             ◇盲◇      ◇小◇
             ◇區◇      ◇隱◇
        │←──┼──────┼──→│
         ◇開◇大      ◇區◇
         ◇放◇      │
         ◇區◇      ◇小◇
                   ◇未◇
                   ◇知◇
```

特點：开放区大，隐藏区和盲区小

**行動**：
- 持續分享
- 定期反饋
- 探索新機會

#### 一對一應用

**擴大開放區**：
- 定期一對一對話
- 分享工作狀況
- 討論個人目標

**縮小盲區**：
- 主動請求反饋
- 觀察他人反應
- 反思自己的行為影響

**縮小隱藏區**：
- 分享必要資訊
- 降低過度防禦
- 建立信任關係

### 2.2 認知／行動循環

#### 循環結構

```
     ┌─────────┐
     │  現況   │ ←┐
     │ Status  │  │循環
     └────┬────┘  │
         │ 問：  │
         │「為什  │
         │  這樣？」│
         ▼       │
     ┌─────────┐  │
     │  認知   │  │
     │  問題   │  │
     └────┬────┘  │
         │ 問：  │
         │「實際  │
         │ 狀況如 │
         │ 何？」  │
         ▼       │
     ┌─────────┐  │
     │  事實   │  │
     │ Reality │  │
     └────┬────┘  │
         │ 問：  │
         │「想怎  │
         │   樣？」│
         ▼       │
     ┌─────────┐  │
     │  目標   │  │
     │  Goal   │  │
     └────┬────┘  │
         │ 問：  │
         │「怎麼  │
         │   做？」│
         ▼       │
     ┌─────────┐  │
     │  行動   │  │
     │ Action  │  │
     └────┬────┘  │
         │ 問：  │
         │「結果  │
         │ 如何？」│
         └─────────┘
```

#### 各階段說明

**現況（Status）**
- 開始的當前狀態
- 問題：「為什麼是這樣的現況？」
- 目的：理解最初的狀態

**認知問題（Recognition）**
- 識別問題或機會
- 問題：「實際的狀況如何？」
- 目的：將問題具體化

**事實（Reality）**
- 收集事實數據
- 問題：「數據和事實說明什麼？」
- 目的：客觀評估

**目標（Goal）**
- 設定改進目標
- 問題：「我們想要達成什麼？」
- 目的：明確方向

**行動（Action）**
- 執行改進措施
- 問題：「如何達成目標？」
- 目的：制定具體行動

**結果（Response）**
- 評估行動結果
- 問題：「結果如何？有達成目標嗎？」
- 目的：評估效果，循環改進

#### 應用範例

**場景：提升團隊會議效率**

```
現況：會議經常超時
  ↓ 問：為什麼超時？
認知：沒有明確議程和時間管理
  ↓ 問：實際狀況如何？
事實：60%會議超過預定時間，平均延遲30分鐘
  ↓ 問：想怎樣？
目標：90%會議按時結束
  ↓ 問：怎麼做？
行動：制定議程模板、設議時限制、主持人訓練
  ↓ 問：結果如何？
結果：80%會議按時結束，持續改進
  ↓ 回到現況，循環繼續
```

### 2.3 Want / Commitment

#### 核心區別

| 維度 | Want（想要） | Commitment（承諾） |
|------|--------------|---------------------|
| 態度 | 意願、期望 | 責任、約定 |
| 行為 | 口頭支持 | 持續行動 |
| 穩定性 | 容易變動 | 穩定堅持 |
| 約束力 | 低 | 高 |

#### W/C 矩陣

```
             高 Want
         ┌───────────┐
         │ 完美狀態 │
  低Commitment   │ ✓/✓ │
         └───────────┘

    ┌────┴────────────┐
    │                 │
 ┌──┴──┐          ┌──┴──┐
 │W/✗ │          │C/✓ │
 │無願│          │責任│
 │有承│          │驅動│
 └────┘          └────┘
         ┌───────────┐
         │  問題狀態│
         │   ✗/✗   │
         └───────────┘
         低 Want
```

#### 四種狀態與應對

**完美狀態（Want √, Commitment √）**
- 定義：有意願且有承諾
- 行動：主動參與、持續推動
- 管理策略：充分授權、挑戰成長

**無願有承（Want ✗, Commitment √）**
- 定義：沒有意願但有承諾
- 行動：責任驅動、按章辦事
- 管理策略：激發動機、溝通價值
- 問題句：
  - 「這個任務對你有什麼意義？」
  - 「達成目標後你會有什麼收穫？」

**有願無承（Want √, Commitment ✗）**
- 定義：想要但沒有明確承諾
- 行動：口頭支持、行動不穩
- 管理策略：建立約定、追蹤進度
- 問題句：
  - 「你願意承諾做什么？」
  - 「我們可以如何設定明確的約定？」

**問題狀態（Want ✗, Commitment ✗）**
- 定義：沒有意願也沒有承諾
- 行動：被動、不投入
- 管理策略：評估是否需要、重新分配

#### 轉化技巧

**Want 為主（提升 Commitment）**
1. 對話了解為什麼 Want 不等同 Commitment
2. 確認障礙或擔憂
3. 制定明確的約定和時限
4. 同意監督機制
5. 確責任人

**Commitment 為主（提升 Want）**
1. 了解成為 Commitment 的原因
2. 驗證價值和意義
3. 連接個人目標
4. 提供正向反饋

### 2.4 PM理論（Positive Mentoring）

#### 三個要素

| 要素 | 說明 | 行動 |
|------|------|------|
| Positive（正向） | 正面態度 | 專注優點、給予鼓勵 |
| Model（模式） | 模範行為 | 以身作則、展示期待 |
| Mentoring（導師） | 導師關係 | 指導成長、建立信任 |

#### Positive（正向）

**原則**：
- 專注優點而非缺點
- 情境正向思考
- 給予正向反饋

**行動方式**：
```
原來：「你報遲了，會影響其他成員。」
正向：「我注意到你報遲了，下次我們可以一起確保準時嗎？」

原來：「這個方案做得不好。」
正向：「我在方案中看到一些好想法，我們可以再一起優化嗎？」
```

**正向反饋技巧**：
1. 描述行為
2. 解釋影響
3. 表達感謝

範例：「你在會議中主動分享研究結果（行為），幫助團隊更好地理解問題（影響），謝謝你的貢獻（感謝）。」

---

#### Model（模式）

**原則**：
- 以身作則
- 展示期待
- 堅持標準

**行動範例**：
- 如果你希望成員準時先準時
- 如果你希望成員溫溝通也要溫柔溝通
- 如果你希望成員持續改進就也要持續改進

「做你期望看到的事」

---

#### Mentoring（導師）

**核心價值**：
- 一對一關注成員成長
- 提供具體指導
- 建立長期信任關係

**一對一會議結構**：
1. 開場（5分鐘）：建立氛圍
2. 進展評估（10分鐘）：討論工作狀況
3. 挑戰與機會（15分鐘）：討論成長需求和機會
4. 行動規劃（10分鐘）：制定下一步行動
5. 結束（5分鐘）：鼓勵與感謝

#### PM 綜合應用

**情境：團隊成員改進方案**：

**正向**：
「我認為這個改進的願景很好，對團隊是一個很好的機會。」

**模式**：
「我也在嘗試同樣的方式改進我的工作流程。」

**導師**：
「我可以和你一起看看如何實施，你覺得怎樣？」

### 2.5 利害關係人分析

#### 分析框架

| 利害關係人 | 影響程度 | 利益相關性 | 關注重點 | 應對策略 | 頻率 |
|-----------|---------|-----------|---------|---------|------|
| 客戶 | 高 | 高 | 品質、價格、服務 | 溝通滿意度、收集需求 | 每月 |
| 員工 | 高 | 高 | 薪資、發展、環境 | 透明溝通、培訓發展 | 每週 |
| 投資人 | 高 | 高 | ROI、成長、風險 | 定期報告、透明的財務 | 每季 |
| 管理層 | 高 | 高 | 目標達成、效率 | 目標共識、績效管理 | 每月 |
| 供應商 | 中 | 中 | 合作穩定、付款 | 建立夥伴、公平交易 | 每季 |
| 政府 | 中 | 中 | 法規遵循、稅務 | 主動配合、資訊公開 | 每年 |
| 競爭者 | 低 | 低 | 市場份額、趨勢 | 監控分析、參考學習 | 每季 |
| 媒體 | 低 | 低 | 形象、新聞 | 適度溝通、危機處理 | 隨需 |

#### 應用步驟

**步驟1：識別利害關係人**
- 列出所有可能影響組織的人/群體
- 考慮內部和外部

**步驟2：評估影響程度與利益相關性**
- 影響程度：對組織的影響大小
- 利益相關性：與組織利益的關聯程度

**步驟3：識別關注重點**
- 每個利害關係人最關心什麼
- 將價值對具象化

**步驟4：制定應對策略**
- 基於關注重點制定對應策略
- 設定溝通頻率

## STEP 3: 成員動機

### 3.1 雙因素理論

#### 簡介
由弗雷德里克·赫茲伯格（Frederick Herzberg）於 1950 年代提出。

#### 兩類因素

| 類型 | 說明 | 範例 | 作用 |
|------|------|------|------|
| 保健因素 | 預防不滿意 | 工作環境、薪酬、安全、政策 | 沒有會不滿意，有不一定滿意 |
| 激勵因素 | 增加滿意意 | 成就、認可、責任、成長 | 有會滿意，加強動機 |

#### 保健因素（消極因素）

| 範例 | 說明 | 實際應用 |
|------|------|---------|
| 工資福利 | 是否達到市場水平 | 定期評估薪酬競爭力 |
| 工作條件 | 環境、設備、安全 | 提供舒適的工作環境 |
| 公司政策 | 清晰規則和流程 | 建立公平透明的制度 |
| 人際關係 | 與同事主管的關係 | 培養正向團隊文化 |
| 工作保障 | 工作穩定性 | 定期溝通組織穩定性 |

**重點**：
- 保證這些因素存在
- 避免「不滿意」
- 但單靠這些不能提升「滿意度」

#### 激勵因素（積極因素）

| 範例 | 說明 | 實際應用 |
|------|------|---------|
| 成就感 | 完成困難任務 | 設定挑戰性目標並祝賀達成 |
| 認可 | 被肯定與重視 | 公開讚賞、回饋 |
| 工作本身 | 有趣、有意義 | 分配感興趣、有意義的工作 |
| 責任 | 被授予權力 | 授權負責特定工作 |
| 成長 | 學習和進步機會 | 提供培訓、挑戰性項目 |
| 進步 | 職涯發展 | 制定個人發展計畫 |

**重點**：
- 這些因素能真正提升滿意度和動機
- 需要主動創造

#### 應用原則

**先確保保健因素**：
- 評估現狀確保保健因素不會成為問題
- 如果保健因素不佳，先解決

**再加入激勵因素**：
- 保健因素達標後加入激勵因素
- 持續為員工創造成長機會

**實際範例**：

**員工 A**：
- 保健：薪酬合理（✓）
- 激勵：沒有挑戰性工作（✗）
- 結論：不滿但沒有滿意 → 只會按部就班

**措施**：
- 分配挑戰性任務
- 設置進展追蹤
- 設計認可機制

### 3.2 Will / Skill 矩陣

#### 核心概念

依據意願和能力兩個維度，制定不同管理策略。

#### 矩陣結構

```
                     高意願
                ┌──────────────┐
                │   明星型    │
   高能力             │   ✓   ✓      │
                │   放手授權   │
   ┌─────────────┼──────────────┤
   │             │   責任型    │
   │             │   ✓   ✗      │
   │             │   確認激勵   │
   └─────────────┼──────────────┘
                │   跟進型    │
                │   ✗   ✓      │
                │   培訓支援   │
                └──────────────┘
                  低意願
```

#### 四種類型與應對

**明星型（高能力 + 高意願）**
- 定義：有實力又有意願
- 特點：自發完成任務、願意挑戰
- 管理策略：
  - 充分授權
  - 挑戰成長
  - 擔任導師角色
  - 分享重要機會

**跟進型（低能力 + 高意願）**
- 定義：有熱情但經驗不足
- 特點：願意學、積極但欠缺方法
- 管理策略：
  - 提供培訓
  - 指派導師
  - 給予具體指導
  - 關注進展

**責任型（高能力 + 低意願）**
- 定義：有實力但熱情不夠
- 特點：能做好但不想投入
- 管理策略：
  - 了解原因
  - 確認重要性
  - 重新規劃工作
  - 提供激勵

**問題型（低能力 + 低意願）**
- 定義：無實力也無熱情
- 特點：不積極也不達標
- 管理策略：
  - 評估是否需要
  - 確定目標
  - 可能調整或離開

#### 轉化路徑

**跟進型 → 明星型**（能力提升）
- 提供培訓
- 導師輔導
- 經驗積累

**責任型 → 明星型**（意願提升）
- 了解熱情不夠原因
- 連接價值與意義
- 適當激勵

### 3.3 GROW 模型

#### 簡介
一對一輔導的結構化方法，源於 John Whitmore。

#### 四個階段

| 字母 | 英文 | 中文 | 詢問重點 |
|------|------|------|----------|
| G | Goal | 目標 | 你想要達成什麼？ |
| R | Reality | 現實 | 目前的情況如何？ |
| O | Options | 方案 | 有哪些可行的方案？ |
| W | Will | 行動 | 你打算怎麼做？ |

#### 詳細解說

**Goal（目標）**
- 協助受輔導者明確目標
- 確保目標具體、可衡量
- 認清這個目標的重要性

**Reality（現實）**
- 了解現狀
- 分析障礙與資源
- 客觀評估現有能力

**Options（方案）**
- 腦力激盪所有可能的方案
- 不批判、不過濾
- 協助選擇最適當的方案

**Will（行動）**
- 確定行動計畫
- 設定時間點
- 確定如何追蹤進展

#### 應用範例

**情境：改進團隊效率**

**Goal**：
「你在團隊效率方面想要達成什麼目標？」
「我想要會議時間縮少 30%。」

**Reality**：
「目前會議的狀況如何？你每天開多長時間會議？」
「平均每天 2 小時，但經常超時。」

**Options**：
「我們可以如何改善會議效率？有什麼方案？」
- 設定更嚴格的議程
- 縮短會議時長
- 使用線上工具
- 精簡參與人員

**Will**：
「你打算實施哪個方案？什麼時候開始？」
「我先從議程開始，下週開始執行。」
