# 分类板块详解

## 现有六大板块
- 🪶 Remember This — 箴言/原则(1–3 条;可斜体)
- 🤓 Fav Finds — 当日最值得收藏(概念式两行/图标单行)
- 📘 Read This — 文章/长文/官方博客(概念/要点/官方博客样式)
- 🛠️ Tools — 产品/服务/CLI/代码仓库(图标单行/概念式/GitHub Repo)
- 🔧 Try This — 教程/实验/可实践方法(概念式两行/要点式)
- 🎧 Listen To — 播客/音频(概念式两行)

## 分类映射规则
```
内容类型                    → 目标板块
─────────────────────────────────────────
官方博客/技术文章            → 📘 Read This
开源项目/工具/产品           → 🛠️ Tools
方法论/思考/原则             → 🤓 Fav Finds 或 🪶 Remember This
教程/实验/代码示例           → 🔧 Try This
播客/音频内容               → 🎧 Listen To
值得深挖的生态资源           → 🤓 Fav Finds
```

## 去重策略
- 同一 URL 在同一天仅记录一次
- 重复出现时合并来源信息与描述
- 保留最完整的上下文


## 增加新板块指引
- 在 docs/daily-links 模板中添加对应标题与图标
- 为新板块定义映射与适用样式(概念/要点/官方/Repo/图标单行)
- 在 SKILL.md 的“分类映射规则”与“输出样式详解”补充说明
- 如需专用样式,在 references 目录新增对应样式文档
