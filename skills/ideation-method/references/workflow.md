## 阶段一：创意激发详细流程

### 1.1 准备阶段
> 时间：5-10分钟

| 步骤 | 动作 | 注意事项 |
|------|------|----------|
| 1 | 明确课题 | 用一句话描述要解决的问题 |
| 2 | 定义边界 | 明确什么在范围内，什么不在 |
| 3 | 组建团队 | 确保多元背景（可选） |
| 4 | 准备工具 | 白板、便签纸、计时器 |

### 1.2 腦力書寫执行
> 时间：10-15分钟

| 时间 | 动作 |
|------|------|
| 0-2分 | 热身：快速写下最初3-5个想法 |
| 2-12分 | 自由书写：不设限制，疯狂想法欢迎 |
| 12-15分 | 搭顺风车：基于他人想法延伸 |

### 1.3 曼陀羅九宮格执行
> 时间：10-15分钟

1. 在九宫格中心写下核心主题
2. 按八个方向依次思考
3. 每个格子至少1个创意

### 1.4 型態分析执行
> 时间：10分钟

1. 确定关键维度（通常3-5个）
2. 为每个维度列出选项（每个至少3个）
3. 组合不同维度的选项生成新创意

### 1.5 腳本圖推演
> 时间：10分钟

1. 定义关键不确定因素
2. 描述三种情境
3. 识别决策点

### 1.6 奧斯本檢核表
> 时间：10分钟

1. 逐个进行七方向检核
2. 每个方向至少1个创意
3. 记录所有产生的想法

### 1.7 创意整合
> 时间：5分钟

| 操作 | 说明 |
|------|------|
| 去重 | 删除完全相同的创意 |
| 合并 | 将相似创意合并为一个条目 |
| 分类 | 按类型或方法分组 |
| 筛选 | 保留最具潜力的创意（目标>10个） |

## 阶段二：创意具象化详细流程

### 2.1 创意表单填写
> 时间：每个创意5-10分钟

按以下顺序填写：
1. 创意名称 → 一句话概括
2. 核心概念 → 价值主张
3. 目标用户 → 明确受益者
4. 核心功能 → 3-5个关键点
5. 独特优势 → 与现有方案差异
6. 资源需求 → 人力/时间/预算
7. 风险点 → 潜在困难
8. 成功指标 → 可衡量结果

### 2.2 流程图绘制
> 时间：每个创意5分钟

1. 识别关键节点
2. 确定节点顺序
3. 标注决策点
4. 注明所需资源

---

## 阶段三：创意评比详细流程

### 3.1 優缺點評比
> 时间：每个创意3-5分钟

1. 列出至少3个优点
2. 列出至少3个缺点
3. 确保描述具体
4. 附上影响程度评估

### 3.2 SUCCESs 評估
> 时间：每个创意5分钟

1. 逐个维度评估（1-5分）
2. 填写评分理由
3. 计算总分
4. 排序

### 3.3 報酬矩陣評估
> 时间：每个创意3分钟

1. 评估報酬（高/低）
2. 评估機率（高/中/低）
3. 确定类型
4. 给出建议

### 3.4 综合排名
> 时间：5分钟

结合三种评估结果，给出最终排名

## 团队协作流程

### 角色分配

| 角色 | 职责 | 人数 |
|------|------|------|
| 主持人 | 引导流程、控制时间 | 1 |
| 记录员 | 记录想法、整理资料 | 1 |
| 参与者 | 提供想法、参与评比 | 3-7 |

### 团队工作节奏

```
阶段1: 创意激发（集体）
  ↓
   [休息 10分钟]
  ↓
阶段2: 创意具象化（分组）
  - A组: 奇数创意
  - B组: 偶数创意
  ↓
阶段3: 创意评比（集体匿名投票）
  ↓
  生成报告
```

## 成見防范关键检查点

### 检查点1: 创意激发后
> 在进入 STEP 2 之前

```
检查清单:
  [ ] 创意数量是否 > 10 个？
  [ ] 是否使用了至少 3 种激发方法？
  [ ] 是否有"疯狂想法"被记录？
  [ ] 初始想法是否未被特殊对待？
```

### 检查点2: 创意具象化后
> 在进入 STEP 3 之前

```
检查清单:
  [ ] 每个创意的优缺点是否等量？
  [ ] 是否使用数据而非模糊描述？
  [ ] 是否邀请了不同背景人员参与？
```

### 检查点3: 最终决策前
> 在生成报告之前

```
检查清单:
  [ ] 是否进行了匿名评比？
  [ ] 反对意見是否得到平等权重？
  [ ] 是否忽略了最初的几个想法？
  [ ] 决策基于多種评估方法的綜合結果？
```

## 常见问题处理

### Q1: 创意生成数量不足
**解决方案**:
- 延长时间再进行5分钟腦力書寫
- 专门使用曼陀羅九宮格进行补充
- 邀请更多人参与

### Q2: 创意质量不高
**解决方案**:
- 检查是否受到成見影响（锚定效应）
- 鼓励"疯狂想法"
- 使用型態分析法进行系统组合

### Q3: 评比结果不明确
**解决方案**:
- 檢查成見是否影响了判断
- 增加評估维度
- 使用數據驗證直覺判斷

### Q4: 意見分歧严重
**解决方案**:
- 使用匿名投票
- 引入外部評估者
- 使用多種評估方法的綜合結果
